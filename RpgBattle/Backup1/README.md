# RPGバトルひな型（C#・オブジェクト指向学習用）

コンソール（CUI）で動くターン制 RPG バトルの学習用ひな型です。  
**カプセル化・継承・ポリモーフィズム**を体感しながら、約5時間程度の演習で拡張できます。

## 動かし方

```bash
cd RpgBattle
dotnet run
```

ビルドのみ確認する場合:

```bash
dotnet build
```

起動後、メニューで `1`（戦う） / `2`（回復する） / `3`（逃げる） を入力してください。

## ファイル構成

| ファイル | 内容 | 受講者の作業 |
|----------|------|--------------|
| `Character.cs` | 基底クラス（カプセル化済み） | コメントのヒントに沿ってステータスやメソッドを追加 |
| `Warrior.cs` | 戦士（Attack の override 例） | クリティカルなど攻撃のアレンジ／**新しい職業クラスの追加** |
| `Mage.cs` | 魔法使い（Attack の override 例） | MP 消費など魔法のアレンジ |
| `Program.cs` | 戦闘ループ（入力・勝敗判定） | 敵 AI やキャラ差し替え |

## 学習の進め方（おすすめ順）

1. **動かして理解** — `dotnet run` で戦い、ログの「剣での攻撃」「火の玉」の違いを確認する  
2. **カプセル化** — `Character.cs` の private フィールドと `TakeDamage` / `Heal` を読む。値を外側から直接いじれない理由を考える  
3. **継承でキャラ増加** — `Warrior.cs` 末尾の課題コメントを見ながら `Archer` などを新規作成する（※課題パートは未実装）  
4. **ポリモーフィズム** — `Program.cs` の `player.Attack(enemy)` が、実際の型に応じて別メッセージになることを確認する。自分の職業でも override する  
5. **戦闘ループ拡張** — 敵の AI、回復回数制限、クリティカルなどを足す  

## 最初から動く内容

- 勇者（Warrior） vs スライム魔術師（Mage）のターン制バトル  
- 戦う / 回復する / 逃げる（50%成功）  
- HP が 0 になるまでループし、勝敗を表示  

新しいキャラクタークラスの追加は意図的に空欄（コメントのヒントのみ）にしています。手元で実装してみてください。
